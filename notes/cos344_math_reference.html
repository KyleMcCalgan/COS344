§1 Image & Colour Mathematics ▾
1.1 Image as a Function
I(x, y) : R → V where R ⊂ ℝ² and V is the set of pixel values
An image is modelled as a 2D function mapping coordinate (x, y) to a colour value V. V depends on image type: greyscale V = ℝ⁺, RGB V = (ℝ⁺)³.
✎ Worked Example

For a 24-bit RGB image (1920×1080):
Domain R = {(x, y) | 0 ≤ x < 1920, 0 ≤ y < 1080}
V = (ℝ⁺)³ — a vector of 3 non-negative reals (R, G, B)

I(960, 540) = (0.2, 0.8, 0.4) → centre pixel is greenish

1.2 Pixel Colour Levels per Channel
possibleLevels(n) = 2^(n/3)
In an n-bit RGB system, each of the 3 channels gets n/3 bits. Levels = 2 raised to that share.
✎ Worked Example

24-bit RGB: possibleLevels(24) = 2^(24/3) = 2^8 = 256

R: 256 values, G: 256 values, B: 256 values → 256³ = 16,777,216 total colours.

1.3 Pixel Colour Levels — with Alpha
possibleLevelsAlpha(n) = 2^(n/4)
In an RGBA system, n bits are shared equally among 4 channels.
✎ Worked Example

32-bit RGBA: possibleLevelsAlpha(32) = 2^(32/4) = 2^8 = 256

Each of R, G, B, A can take values 0–255.

1.4 Alpha Compositing (Pixel Blending)
c = α · c_f + (1 − α) · c_b
Variables
cResulting blended colour (vector)
c_fForeground pixel colour (vector)
c_bBackground pixel colour (vector)
αCoverage fraction (0 = transparent, 1 = opaque)
Linearly interpolates between foreground and background based on transparency. α=1 → foreground replaces background. α=0 → background unchanged.
✎ Worked Example

c_f = (0, 1, 1) [cyan], c_b = (1, 1, 0) [yellow], α = 0.3

c = 0.3 × (0, 1, 1) + 0.7 × (1, 1, 0)
  = (0.0, 0.3, 0.3) + (0.7, 0.7, 0.0)
  = (0.7, 1.0, 0.3)  ← brownish-green
§2 Ray Tracing Mathematics ▾
2.1 Parametric Ray Equation
p(t) = e + t(s − e) = e + t · d where d = s − e
Variables
p(t)Point on the ray at parameter t
eEye / viewpoint (ray origin)
sA point the ray passes through
dDirection vector of the ray
tReal-valued parameter
p(0) = e (at the eye). p(1) = s (at the image plane). If t₁ < t₂ then p(t₁) is closer to e.
✎ Worked Example

Camera at e = (0,0,5), image plane point at s = (1,0,0), find p at t = 0.5:

d = (1,0,0) − (0,0,5) = (1, 0, −5)
p(0.5) = (0,0,5) + 0.5×(1,0,−5)
       = (0,0,5) + (0.5, 0, −2.5)
       = (0.5, 0, 2.5)
2.2 Pixel → Image-Plane Mapping
u = l + (r − l)(i + 0.5) / nx v = b + (t − b)(j + 0.5) / ny
Variables
(i, j)Pixel coordinates (column, row)
nx, nyImage width and height in pixels
l, rLeft and right edges of image plane
b, tBottom and top edges of image plane
(u, v)Continuous coordinates on image plane
The + 0.5 centres the sample at the pixel's midpoint (not its corner).
✎ Worked Example

Image: 800×600, plane: l=−1, r=1, b=−0.75, t=0.75. Pixel (400, 300):

u = −1 + 2 × 400.5/800
  = −1 + 1.00125 ≈ 0.00125  (≈ centre ✓)

v = −0.75 + 1.5 × 300.5/600
  ≈ 0.00125  (≈ centre ✓)
2.3 Orthographic Viewing Ray
Ray origin = e + u·û + v·v̂ Ray direction = −ŵ
All rays are parallel — same direction (−ŵ), different origins shifting across the image plane.
2.4 Perspective Viewing Ray
Ray origin = e Ray direction = −d·ŵ + u·û + v·v̂
All rays originate from the same point (the eye). Direction fans out per pixel. d is the focal length.
✎ Worked Example

Camera at e=(0,0,0), û=(1,0,0), v̂=(0,1,0), ŵ=(0,0,1), d=1, u=0.5, v=0.3:

dir = −1×(0,0,1) + 0.5×(1,0,0) + 0.3×(0,1,0)
    = (0,0,−1) + (0.5,0,0) + (0,0.3,0)
    = (0.5, 0.3, −1)  → normalise for unit vector
2.5 Ray–Sphere Intersection
(d·d)t² + 2d·(e−c)t + (e−c)·(e−c) − R² = 0
Quadratic A, B, C
Ad · d
B2d · (e−c)
C(e−c)·(e−c) − R²
D < 0 → No intersection (misses sphere)
D = 0 → Tangent (one intersection)
D > 0 → Two intersections — take smallest positive t
✎ Worked Example

Sphere at c=(0,0,−3), R=1. Ray from e=(0,0,0) direction d=(0,0,−1):

A = d·d = 1
e−c = (0,0,3)
B = 2×(−1)(3) = −6
C = 9 − 1 = 8

D = (−6)² − 4(1)(8) = 36 − 32 = 4  → D > 0: two hits
t = (6 ± 2)/2  →  t₁=2, t₂=4

Take t=2. Hit point: p(2) = (0, 0, −2) → front of the sphere.

2.6 Ray–Triangle Intersection (Barycentric)
e + td = a + β(b−a) + γ(c−a)
Gives a 3×3 linear system in unknowns t, β, γ. Solved with Cramer's Rule.
Hit iff: β > 0, γ > 0, and β + γ < 1
✎ Worked Example

Triangle: a=(0,0,0), b=(1,0,0), c=(0,1,0). Ray: e=(0.2,0.2,1), d=(0,0,−1):

x: 0.2 = β  → β = 0.2
y: 0.2 = γ  → γ = 0.2
z: 1−t = 0  → t = 1

β=0.2>0 ✓  γ=0.2>0 ✓  β+γ=0.4<1 ✓  → HIT!
§3 Shading Mathematics ▾
3.1 Viewing Direction
v = −d / ||d||
Unit vector from hit point back toward the camera — negation of the normalised ray direction.
✎ Worked Example

Ray direction d = (1, −1, −2)
||d|| = √(1+1+4) = √6 ≈ 2.449
v = −(1,−1,−2) / 2.449 = (−0.408, 0.408, 0.816)

3.2 Irradiance — Point Light Source
E = I / r²
Light from a point source spreads over the surface of a growing sphere. Area grows with r², so intensity per unit area falls off with 1/r².
✎ Worked Example

I = 100, at r = 5: E = 100/25 = 4 W/m²

Same light at r = 10: E = 100/100 = 1 W/m² (4× dimmer at 2× distance)

3.3 Irradiance with Lambert's Cosine Law
E = I · (n · l) / r² = I · cos(θ) / r²
Variables
nSurface normal (unit vector)
lUnit vector toward the light
θAngle of incidence (n · l = cos θ)
When light hits at an angle, it spreads over a larger area. At grazing angles (θ → 90°), cos(θ) → 0 — almost no light received.
✎ Worked Example

I = 100, r = 5, n = (0,1,0), l = (0, 0.707, 0.707):

n · l = 0.707
E = 100 × 0.707 / 25 ≈ 2.83 W/m²
3.4 Directional Light Irradiance
E = H · cos(θ)
For a directional light (like the sun) the 1/r² falloff is negligible. Irradiance depends only on surface orientation relative to light direction. H is the constant normal irradiance.
3.5 Lambertian (Diffuse) Reflection
Lr = (R / π) · E Full form: Lr = (R / π) · I · max(0, n · l) / r²
Ideal diffuse surface — reflects equally in all directions (matte paint, chalk, rough wood). max(0, n·l) clamps negative values: surface facing away receives zero illumination.
✎ Worked Example

R = (0.8, 0.2, 0.1) (reddish), E = 3.0 W/m²:

Lr = (0.8/π, 0.2/π, 0.1/π) × 3.0
   = (0.765, 0.191, 0.095)  ← predominantly red
3.6 Half Vector (Blinn-Phong)
h = (l + v) / ||l + v||
The half vector bisects the angle between the light direction and the view direction. When h aligns with the surface normal n, specularity is at maximum — this is where the highlight appears.
✎ Worked Example

l = (0,1,0), v = (1,0,0):

l + v = (1, 1, 0)
||l+v|| = √2 ≈ 1.414
h = (0.707, 0.707, 0)
3.7 Phong Specular Term
(n · h)^p
p valueAppearance
1–10Very broad highlight (plastic / rubber)
50–100Medium glossy (polished wood)
500+Very tight highlight (polished metal)
✎ Worked Example

n = (0,1,0), h = (0.1, 0.995, 0), p = 50:

n · h = 0.995
(n · h)^50 = 0.995^50 ≈ 0.778
3.8 Combined Blinn-Phong + Lambertian
Lr = ( R/π + ks · max(0, n·h)^p ) · E
Variables
RDiffuse reflectance colour
ksSpecular coefficient colour
pPhong exponent (shininess)
EIrradiance at surface
✎ Worked Example

R = (0.6,0.6,0.6), ks = (0.4,0.4,0.4), p = 64, E = 2.0, n·h = 0.9:

Specular = (0.9)^64 ≈ 0.001177
Diffuse  = 0.6/π   ≈ 0.191
Lr = (0.191 + 0.4×0.001177) × 2.0
   ≈ 0.383 per channel

When n·h ≈ 1, the specular term dominates.

3.9 Mirror Reflection Direction
r = d − 2(d · n)n
The reflected ray is the incoming ray flipped across the surface normal. (d·n) projects d onto n; subtracting twice this reverses the component perpendicular to the surface.
✎ Worked Example

d̂ = (0.707, −0.707, 0), n = (0,1,0):

d · n = −0.707
r = (0.707, −0.707, 0) − 2(−0.707)(0,1,0)
  = (0.707, −0.707, 0) + (0, 1.414, 0)
  = (0.707, 0.707, 0)  ← bounces up ✓
3.10 Ambient Illumination
Lr = ka · Ia
Variables
kaAmbient reflection coefficient (RGB, 0–1)
IaAmbient light intensity (RGB)
Directionless — fills all shadows uniformly. No geometry (normals, distances) involved.
✎ Worked Example

ka = (0.1, 0.1, 0.1), Ia = (1.0, 1.0, 1.0)
Lr = (0.1, 0.1, 0.1) — dark grey ambient contribution

§4 2D Transformation Matrices ▾
4.1 2D Scaling Matrix
scale(sx, sy) = | sx 0 | | 0 sy |
Multiplies x by sx and y by sy. Uniform if sx = sy; non-uniform (deforms) if sx ≠ sy.
✎ Worked Example

Scale (3, 2) by sx=2, sy=3:

[2  0] [3]   [6]
[0  3] [2] = [6]  → Result: (6, 6)
4.2 2D Shearing Matrices
shearX(s) = | 1 s | shearY(s) = | 1 0 | | 0 1 | | s 1 | Angular: shearX(φ) = | 1 tan(φ) | | 0 1 |
✎ Worked Example

shearX(0.5) on (2, 3):

[1  0.5] [2]   [3.5]
[0    1] [3] = [3  ]  → Result: (3.5, 3)
4.3 2D Reflection Matrices
reflectY = | −1 0 | (flips X, reflects over Y) | 0 1 | reflectX = | 1 0 | (flips Y, reflects over X) | 0 −1 |
✎ Worked Example

Reflect (4, 2) over Y-axis:

[−1  0] [4]   [−4]
[ 0  1] [2] = [ 2]  → Result: (−4, 2)
4.4 2D Rotation Matrix
rotate(φ) = | cos(φ) −sin(φ) | | sin(φ) cos(φ) |
Rotates counter-clockwise by φ. Derived from:
x_b = x_a·cos(φ) − y_a·sin(φ)
y_b = y_a·cos(φ) + x_a·sin(φ)
✎ Worked Example

Rotate (1, 0) by φ = 90°:

[cos90°  −sin90°] [1]   [ 0  −1] [1]   [0]
[sin90°   cos90°] [0] = [ 1   0] [0] = [1]
→ Result: (0, 1) — moved from +X to +Y axis ✓
4.5 2D Translation Matrix (Homogeneous)
translate(xt, yt) = | 1 0 xt | | 0 1 yt | | 0 0 1 | Point: (x, y, 1) Direction: (x, y, 0)
Homogeneous w=1 → point (can be translated). w=0 → direction (translation has no effect). This unifies all 2D transforms into 3×3 matrices.
✎ Worked Example

Translate (3, 1) by (5, −2):

[1  0   5] [3]   [8 ]
[0  1  −2] [1] = [−1]
[0  0   1] [1]   [1 ]  → Result: (8, −1)
4.6 Composition of 2D Transformations
M = T₂ · R · T₁ (read right-to-left)
To rotate about own centre p:
T₁ = translate(−p), R = rotate(φ), T₂ = translate(p)
✎ Worked Example

Rotate (2, 3) by 90° about centre (1, 1):

T₁: (2,3) → (1,2)    [translate by (−1,−1)]
R:  (1,2) → (−2,1)   [rotate 90°]
T₂: (−2,1) → (−1,2)  [translate by (1,1)]

Combined matrix M = T₂ · R · T₁:
[0 −1 2]
[1  0 0]
[0  0 1]

Apply to (2, 3, 1):
0×2 + (−1)×3 + 2 = −1
1×2 + 0×3 + 0 = 2  → (−1, 2) ✓
§5 3D Transformation Matrices ▾
5.1 3D Scaling Matrix
scale(sx, sy, sz) = diag(sx, sy, sz) [sx 0 0 ] [0 sy 0 ] [0 0 sz ]
✎ Worked Example

Scale (2, 3, 4) by (1, 2, 0.5):

[1    0    0  ] [2]   [2]
[0    2    0  ] [3] = [6]
[0    0    0.5] [4]   [2]
5.2 3D Rotation Matrices
Rz(φ) = | cos φ −sin φ 0 | | sin φ cos φ 0 | | 0 0 1 | Rx(φ) = | 1 0 0 | | 0 cos φ −sin φ | | 0 sin φ cos φ | Ry(φ) = | cos φ 0 sin φ | | 0 1 0 | | −sin φ 0 cos φ |
✎ Worked Example

Rotate (1, 0, 0) around Z-axis by 90°:

Rz(90°)×(1,0,0) = (cos90°, sin90°, 0) = (0, 1, 0)

Rotate (0, 1, 0) around X-axis by 90°:

Rx(90°)×(0,1,0) = (0, cos90°, sin90°) = (0, 0, 1)
5.3 3D Translation Matrix (Homogeneous 4×4)
T(d) = | 1 0 0 dx | | 0 1 0 dy | | 0 0 1 dz | | 0 0 0 1 |
✎ Worked Example

Translate (1, 2, 3) by d = (5, −1, 2):

[1  0  0  5] [1]   [6]
[0  1  0 −1] [2] = [1]
[0  0  1  2] [3]   [5]
[0  0  0  1] [1]   [1]  → Result: (6, 1, 5)
5.4 Transforming Normal Vectors
n_new = (M⁻¹)ᵀ · n then normalise: n̂_new = n_new / ||n_new||
Normals represent surface orientations, not positions. Applying M directly would distort them (especially under non-uniform scaling). The inverse transpose preserves orthogonality.
✎ Worked Example

Scale surface by (2, 1, 1). M = diag(2, 1, 1). Normal n = (1, 0, 0):

M⁻¹ = diag(0.5, 1, 1)
(M⁻¹)ᵀ = diag(0.5, 1, 1)
n_new = (0.5, 0, 0)
Normalise: (1, 0, 0) ✓  (unchanged, uniform in this case)
§6 Graphics Pipeline Mathematics ▾
6.1 Implicit Line Equation
f(x,y) = (y₀−y₁)x + (x₁−x₀)y + x₀y₁ − x₁y₀ = 0
Points on the line: f = 0. One side: f > 0. Other side: f < 0. Used in the midpoint rasterization algorithm to decide which pixel to draw next.
✎ Worked Example

Line from (1, 1) to (4, 3):

(y₀−y₁) = −2,  (x₁−x₀) = 3,  x₀y₁−x₁y₀ = −1
f(x,y) = −2x + 3y − 1

Test midpoint (3, 2.5):
f(3, 2.5) = −6 + 7.5 − 1 = 0.5 > 0  → do NOT advance y
6.2 Barycentric Coordinates
p(α, β, γ) = αa + βb + γc with α + β + γ = 1, α = 1 − β − γ Inside: 0 < α, β, γ < 1 Colour: c = αc₀ + βc₁ + γc₂
✎ Worked Example

Triangle a=(0,0), b=(4,0), c=(0,4). Point p=(1,1):

4β = 1 → β = 0.25
4γ = 1 → γ = 0.25
α = 1 − 0.5 = 0.5   → all in (0,1) → INSIDE ✓

c₀=red, c₁=green, c₂=blue:
c = 0.5(1,0,0) + 0.25(0,1,0) + 0.25(0,0,1)
  = (0.5, 0.25, 0.25)  ← reddish-purple
6.3 Indexed Mesh Storage
Naïve: 9nt units Indexed: 3nv + 3nt units nt = number of triangles nv = number of unique vertices
Each triangle has 3 vertices × 3 coordinates = 9 values naïvely. Indexed: store each unique vertex once (3nv) plus 3 indices per triangle (3nt).
✎ Worked Example

Cube: 8 unique vertices, 12 triangles:

Naïve:   9 × 12 = 108 units
Indexed: 3×8 + 3×12 = 24 + 36 = 60 units
         → 44% less storage

Savings are much greater for dense meshes.

§7 Rotation About an Arbitrary Axis ▾
7.1 Normalised Rotation Axis
V = P₁ − P₀ u = V / |V| = (αx, αy, αz)
✎ Worked Example

P₀ = (1,1,1), P₁ = (3,2,1):
V = (2, 1, 0), |V| = √5 ≈ 2.236
u = (0.894, 0.447, 0.000)

7.2 Projection Length d
d = √(αy² + αz²)
d is the length of the projection of u onto the y-z plane. Used to build alignment matrices Rx and Ry that align u with the z-axis before applying Rz(θ).
✎ Worked Example (continued from 7.1)

d = √(0.447² + 0.000²) = √(0.200) ≈ 0.447

7.3 Alignment Rotation Matrices
Rx(θx) = | 1 0 0 0 | | 0 αz/d −αy/d 0 | | 0 αy/d αz/d 0 | | 0 0 0 1 | Ry(θy) = | d 0 −αx 0 | | 0 1 0 0 | | αx 0 d 0 | | 0 0 0 1 |
7.4 Full Arbitrary-Axis Rotation Matrix
R = T(P₀) · Rx(−θx) · Ry(−θy) · Rz(θ) · Ry(θy) · Rx(θx) · T(−P₀)
Strategy (read right-to-left):
  1. T(−P₀) — translate so P₀ is at origin
  2. Rx(θx), Ry(θy) — align u with the z-axis
  3. Rz(θ) — rotate by θ around z
  4. Ry(−θy), Rx(−θx) — undo alignment
  5. T(P₀) — translate back
