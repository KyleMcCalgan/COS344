Chapter 9

Rasterization

The object-order rendering approach used in real-time graphics. Converts geometric primitives (triangles) into pixel fragments using barycentric coordinates, handles hidden surfaces with z-buffering, and applies shading per-vertex or per-fragment.

Section 9.1 — Rasterization

Rasterization is the process of determining which pixels are covered by a geometric primitive (triangle), and computing their attributes (colour, depth, texture coords) by interpolation.

§2.9 / §9.1.2 — Barycentric Coordinates

Any point inside a triangle with vertices a, b, c can be expressed as:

p(α, β, γ) = α·a + β·b + γ·c Constraint: α + β + γ = 1 (where α ≡ 1 − β − γ) Point is INSIDE the triangle iff: 0 < α < 1, 0 < β < 1, 0 < γ < 1 If one component = 0: point is on an edge If two components = 0: point is at a vertex
⚠️
Non-standard basis

Barycentric coordinates use a non-orthogonal basis. Standard linear algebra operations do not behave as expected — see Section 2.9 of the textbook.

Colour Interpolation (Gouraud)

If each vertex has a colour (c0, c1, c2), the colour at any interior point is:

c = α·c0 + β·c1 + γ·c2 This is called Gouraud interpolation (see §9.2.4).

Triangle Rasterization Algorithm

The rule: draw a pixel if and only if the centre of the pixel lies inside the triangle (all barycentric coords ∈ (0,1)).

Algorithm 2 — Brute-Force Triangle Rasterization
for each pixel (x, y) in bounding box do
compute (α, β, γ) for pixel centre
if α ∈ [0,1] and β ∈ [0,1] and γ ∈ [0,1] then
c = α·c0 + β·c1 + γ·c2
drawPixel(x, y, c)
end if
end for

Shared edge problem

If two adjacent triangles share an edge and a pixel centre lands exactly on that edge, which triangle draws it?

  • Options: draw twice (overdraw), don't draw (gap), draw exactly once
  • Textbook solution: use an off-screen point — a point guaranteed to always be on one side of any edge. Whichever triangle "owns" the side containing that point draws the pixel.

Section 9.2 — Operations Before and After Rasterization

§9.2.1 — Simple 2D Drawing

Simplest possible case: vertices are already in pixel coordinates, nothing happens in vertex or fragment stages. Application supplies primitives, rasterizer fills them with solid colour.

§9.2.2 — Minimal 3D Pipeline

For 3D rendering, the vertex stage multiplies each vertex by the composite MVP matrix:

clip_position = Projection · View · Model · vertex_position Where: Model = object → world space (translation, rotation, scale) View = world → camera space (camera transform) Projection = camera → clip space (frustum/orthographic)

The Painter's Algorithm was an early approach to ordering primitives back-to-front, but it fails for intersecting objects and occlusion cycles — hence the z-buffer was developed.

§9.2.3 — Z-Buffer (Hidden Surface Removal)

The z-buffer (depth buffer) solves visibility: for each pixel, keep track of the depth of the closest surface drawn so far.

Z-Buffer Algorithm
initialise z_buffer[all pixels] = +∞
for each fragment (x, y, z) do
if z < z_buffer[x][y] then
z_buffer[x][y] = z
colour[x][y] = fragment_colour
end if // else discard fragment
end for
✅
Z-buffer in OpenGL (Ch 17)

Enabled with glEnable(GL_DEPTH_TEST) and glDepthFunc(GL_LESS). The depth buffer must be cleared each frame with glClear(GL_DEPTH_BUFFER_BIT).

§9.2.4 — Gouraud Shading (per-vertex)

Shading is computed per vertex in the vertex stage. The resulting colour is then interpolated across the triangle using barycentric coordinates during rasterization.

  • Faster (fewer shading computations)
  • Less accurate — specular highlights can disappear between vertices

§9.2.5 — Phong Shading (per-fragment)

Geometric information (normals, positions) is interpolated through rasterization, and shading is computed per fragment in the fragment shader.

  • More accurate — specular highlights computed at each pixel
  • More expensive — shading runs for every fragment
MethodWhere shading happensInterpolatedQuality
Gouraud Vertex stage (vertex shader) Colour values Lower — can miss specular highlights
Phong Fragment stage (fragment shader) Normals & positions Higher — correct specular at every pixel

Section 9.3 — Simple Anti-aliasing

Aliasing is the jagged "staircase" effect on triangle edges caused by the binary nature of pixel sampling. Anti-aliasing smooths edges by taking multiple samples per pixel (MSAA) or filtering the coverage.
