Chapter 3

Raster Images

Everything rendered in computer graphics ultimately ends up as a raster image. Understanding pixels, colour representation, and compositing is foundational.

Connects to → Ch 4: Ray Tracing (outputs pixels) Ch 9: Rasterization (fills pixels) Ch 5: Shading (computes pixel colour) Ch 17: Framebuffer in OpenGL

Images, Pixels and Geometry §3.2

A raster image is a rectangular 2D array of pixels (picture elements). This is the output format for every rendering algorithm in COS344.

The underlying data structure

A raster image is stored as a 2D array. For an image of width nx and height ny, the array has dimensions ny × nx (rows × columns), where each cell holds a colour value.

Raster image coordinate system
  (0,ny-1)  ┌───┬───┬───┬───┐  (nx-1, ny-1)
            │   │   │   │   │
            ├───┼───┼───┼───┤
            │   │   │   │   │
            ├───┼───┼───┼───┤
            │   │   │   │   │
  (0, 0)   └───┴───┴───┴───┘  (nx-1, 0)

  Each cell = one pixel
  Origin is typically bottom-left (OpenGL convention)
  Array index: pixel[row][col]  where  row=0..ny-1, col=0..nx-1
💡
Raster devices

Output: monitors (LCD/OLED), projectors, printers. Input: digital cameras, scanners, image sensors. All of these operate on rectangular grids of colour samples.

RGB Colour Model §3.3

Colour in a raster image is represented by mixing different intensities of Red, Green, and Blue light. This matches how the human eye perceives colour (three types of cone cells).

8-bit per channel
0 – 255 per channel
Most common. Total 16.7 million colours. Storage: 3 bytes/pixel (24-bit colour).
Floating-point
[0.0, 1.0] per channel
Used internally for shading calculations. OpenGL uses this range for colour values.
Bit depth
2^n levels per channel
8-bit gives 256 levels. Higher bit depths (16-bit, 32-bit) used in HDR imaging.

Examples

RGB Values (float)
Red     = (1.0, 0.0, 0.0)
Green   = (0.0, 1.0, 0.0)
Blue    = (0.0, 0.0, 1.0)
White   = (1.0, 1.0, 1.0)
Black   = (0.0, 0.0, 0.0)
Yellow  = (1.0, 1.0, 0.0)   // R + G
Cyan    = (0.0, 1.0, 1.0)   // G + B
Magenta = (1.0, 0.0, 1.0)   // R + B

Alpha Compositing §3.4

The alpha channel (α) represents opacity/transparency. It allows compositing — layering one image over another.

The compositing formula (Porter-Duff "over" operation)

C_result = α · C_foreground + (1 − α) · C_background Where: C = colour value (per channel: R, G, B) α = alpha value of the foreground [0.0 = fully transparent, 1.0 = fully opaque]
α valueEffectResult
α = 1.0Fully opaqueForeground completely covers background
α = 0.0Fully transparentBackground shows through completely
α = 0.550% transparent50/50 blend of foreground and background
✅
RGBA images

Images that store alpha have 4 channels: Red, Green, Blue, Alpha. PNG supports RGBA. JPEG does not have an alpha channel.

Gamma Correction

Display hardware (monitors) doesn't display intensity linearly — it applies a power function. Gamma correction compensates for this non-linearity so that stored pixel values correspond to perceptually linear brightness.

Display output ∝ voltage^γ (typically γ ≈ 2.2) To correct: store value = linear_intensity^(1/γ) When display applies ^γ, the result is the correct linear intensity.
⚠️
Shading happens in linear space

All lighting/shading calculations (Ch 5) must be done in linear light space, not gamma-encoded space. Convert to gamma-space only at the final output step.
