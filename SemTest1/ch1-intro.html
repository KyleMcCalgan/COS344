Introduction to Computer Graphics

What computer graphics is, where it's used, how rendering works conceptually, and how graphics APIs are structured — the foundation for everything else in COS344.

Connects to → Ch 4: Ray Tracing (image-order rendering) Ch 9: Rasterization (object-order rendering) Ch 17: OpenGL API

What is Computer Graphics?

Computer graphics is the field of creating, manipulating, and rendering visual content using computers. It encompasses everything from generating 2D images to simulating 3D scenes with realistic lighting.

Applications

  • Video games — real-time 3D rendering
  • Cartoons & animated films — pre-rendered CG imagery
  • Visual effects (VFX) — compositing CG with live footage
  • CAD/CAM — engineering & manufacturing visualisation
  • Medical imaging — MRI/CT scan visualisation
  • Simulations — physics, training, scientific modelling
  • Information visualisation — data representation
  • User interaction — GUI rendering

The Two Rendering Paradigms § core concept

This is one of the most fundamental distinctions in graphics — understanding the difference will help you see how Chapters 4 and 9 relate to each other.

🎯
Exam-likely question

Know the definitions of both rendering approaches and be able to give an example of each (Ray Tracing = image-order; Rasterization = object-order).

Approach How it works Loop structure Example
Image-order rendering Each pixel is considered in turn. For each pixel, all objects that influence it are found and the colour is computed. for each pixel → find intersecting objects Ray Tracing (Ch 4)
Object-order rendering Each object is considered in turn. For each object, all pixels it influences are found and updated. for each object → find influenced pixels Rasterization (Ch 9)

Graphics APIs §1 context

A graphics program typically uses two APIs working together:

  1. Graphics API — to draw things (e.g. OpenGL, Direct3D)
  2. User Interface (UI) API — to handle window creation & input (e.g. GLFW)

API paradigms

ParadigmDescriptionExamples
Integrated Graphics and UI toolkits are built-in features of the language/platform. Java (Swing/AWT)
Library Graphics and UI APIs are separate, provided by external libraries. OpenGL + GLFW in C++
💡
COS344 uses OpenGL 3.3 with C++

Specifically OpenGL 3.3 Core Profile. GLFW is used for window management. OpenGL follows the Library paradigm — it's a C library you link against, separate from the OS window system.

The Graphics Pipeline (Overview)

Modern computers have optimised graphical pipelines. The three fundamental operations are:

  1. Map 3D coordinates to 2D screen coordinates — projection (see Ch 7)
  2. Shade polygons to appear realistic (see Ch 5)
  3. Render polygons in the correct order — depth/visibility (see Ch 9)
