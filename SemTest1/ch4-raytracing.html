Chapter 4

Ray Tracing

The image-order rendering approach. For each pixel, cast a ray and find what it hits. Covers the algorithm, projection types, ray generation, and ray-object intersection.

Connects to → Ch 1: Image-order rendering paradigm Ch 4.5/5: Shading at hit point Ch 3: Output to pixel array

Rendering Refresher § intro

Rendering is the process of taking a set of 3D objects and producing a 2D array of pixels as output.

Ray Tracing is an image-order approach: iterate over pixels, and for each pixel determine which objects influence it.

Section 4.1 — Basic Ray-Tracing Algorithm

A ray tracer determines, for each pixel, which objects are "seen" through that pixel by testing whether a viewing ray intersects any objects.

📐
Viewing ray

A line that emanates from the viewpoint, in the direction the pixel is looking. The ray tracer finds the closest object the ray intersects (unless the object is translucent).

Three components of a ray tracer

  1. Ray generation — compute origin and direction of each pixel's viewing ray based on camera geometry.
  2. Ray intersection — find the closest object that intersects the viewing ray.
  3. Shading — compute pixel colour from the intersection result (normal, light, hit point).
Algorithm 1 — Basic Ray Tracing
for each pixel (i, j) do
ray = computeViewingRay(i, j)
hit = findFirstIntersection(ray, objects)
if hit then
n = computeSurfaceNormal(hit)
color[i][j] = shade(n, light, hit.point)
else
color[i][j] = backgroundColor
end if
end for

Section 4.2 — Perspective

We need to project 3D scenes onto a 2D image plane. Multiple approaches exist:

TypeHow it worksProperty
Parallel (Orthographic) 3D points projected parallel to a projection direction onto the image plane. No convergence — parallel lines stay parallel. No depth foreshortening.
Perspective All rays converge at the viewpoint (eye position). Objects further away appear smaller. Mimics human eye / camera.
💡
Linear Perspective

The most common approach in computer graphics. 3D objects are projected onto an image plane so that straight lines in the scene become straight lines in the image.

Section 4.3 — Computing Viewing Rays

A ray is defined as: p(t) = e + t·d, where e is the ray origin (eye/viewpoint) and d is the ray direction.

To set up the camera coordinate frame, compute orthonormal basis vectors u, v, w:

w = −normalize(g) // g = gaze direction; w points AWAY from scene u = normalize(t × w) // t = "up" vector v = w × u Where e = eye/camera position

4.3.1 — Orthographic (Parallel) Viewing Rays

All rays have the same direction but different origins (one per pixel).

For pixel (i, j) mapped to image coords (u, v): Ray origin = e + u·u_vec + v·v_vec Ray direction = −w
Algorithm 2 — Orthographic Ray Generation
for each pixel (i, j) do
// compute u, v in image plane
u = l + (r − l)(i + 0.5) / nx
v = b + (t − b)(j + 0.5) / ny
ray.origin = e + u·u_hat + v·v_hat
ray.direction = −w_hat
end for

4.3.2 — Perspective Viewing Rays

All rays share the same origin (the viewpoint), but have different directions.

Ray origin = e Ray direction = −d·w + u·u_vec + v·v_vec Where d = plane distance (focal length)
Algorithm 3 — Perspective Ray Generation
for each pixel (i, j) do
u = l + (r − l)(i + 0.5) / nx
v = b + (t − b)(j + 0.5) / ny
ray.origin = e
ray.direction = −d·w_hat + u·u_hat + v·v_hat
end for
🎯
Know the difference for the exam

Orthographic: same direction, different origins. Perspective: same origin (e), different directions. The focal length d appears only in perspective rays.

Section 4.4 — Ray-Object Intersection

Given a ray p(t) = e + t·d, find the value of t at which it hits an object. The smallest positive t gives the closest hit.

4.4.1 — Ray-Sphere Intersection

For a sphere centred at c with radius R, substitute the ray equation into the sphere equation:

Sphere surface: |p − c|² = R² Substitute p = e + t·d: (d·d)t² + 2d·(e−c)t + (e−c)·(e−c) − R² = 0 This is a quadratic in t: at² + bt + c = 0 a = d·d b = 2d·(e−c) c = (e−c)·(e−c) − R² discriminant = b² − 4ac If discriminant < 0: no intersection (ray misses sphere) If discriminant = 0: one intersection (tangent) If discriminant > 0: two intersections (pick smallest positive t)

4.4.2 — Ray-Triangle Intersection

For a triangle with vertices a, b, c, express a point inside the triangle using barycentric coordinates and solve:

e + t·d = a + β(b−a) + γ(c−a) Rearranged into a 3×3 linear system in unknowns t, β, γ. Solve using Cramer's Rule. Hit is INSIDE the triangle if: β > 0, γ > 0, and β + γ < 1
Algorithm 4 — Early-Termination Ray-Triangle Intersection
function intersectsTriangle(ray, a, b, c, t0, t1)
Compute t
if t < t0 or t > t1 then return false
Compute γ
if γ < 0 or γ > 1 then return false
Compute β
if β < 0 or β + γ > 1 then return false
return true
end function
✅
Barycentric coordinates also used in Ch 9

The same (β, γ) coordinates used here for intersection appear in Chapter 9 rasterization for determining whether a pixel centre is inside a triangle and interpolating colours.

Practical Example: Ray Tracing a Sphere

C++ — Ray-Sphere Intersection
struct Ray { vec3 origin, direction; };
struct Sphere { vec3 center; float radius; };

// Returns t of intersection, or -1 if miss
float raySphereIntersect(Ray r, Sphere s) {
    vec3 oc = r.origin - s.center;
    float a = dot(r.direction, r.direction);
    float b = 2.0f * dot(oc, r.direction);
    float c = dot(oc, oc) - s.radius * s.radius;
    float discriminant = b*b - 4*a*c;

    if (discriminant < 0) return -1.0f;  // miss

    // Return closest (smallest positive) t
    return (-b - sqrt(discriminant)) / (2.0f * a);
}
