Chapter 7

Transformations

Matrices are the backbone of 3D graphics — every object position, rotation, and scaling is a matrix multiplication. This chapter covers 2D and 3D linear transforms, homogeneous coordinates, and inverses.

Connects to → Ch 9: Model/camera/projection matrices Ch 17: Matrices in GLSL vertex shaders Ch 5: Normal vector transforms

Why Matrices? § motivation

Instead of applying separate formulas to each vertex, we pack a transformation into a matrix and apply it with a single matrix-vector multiplication. This enables:

  • Composing multiple transforms: M = M3 · M2 · M1
  • Passing transforms efficiently to the GPU
  • Inverting transforms easily
Algorithm — Applying a Transformation Matrix
function applyTransform(Matrix T, Object o)
vertices = o.getVertices()
for each v in vertices do
v' = T · v // matrix × vector
updateVertex(o, v, v')
end for
draw(o.getVertices())
end function

Section 7.1 — 2D Linear Transformations

A linear transformation maps a 2D vector to another 2D vector via matrix multiplication:

[a11 a12] [x] [a11·x + a12·y] [a21 a22] [y] = [a21·x + a22·y]

7.1.1 — Scaling

scale(sx, sy) = [sx 0 ] [ 0 sy] Result: x' = sx·x, y' = sy·y If sx == sy: uniform scale (shape preserved) If sx != sy: non-uniform scale (object deformed/stretched)

7.1.2 — Shearing

shear_x(s) = [1 s] shear_y(s) = [1 0] [0 1] [s 1] shear_x: x' = x + s·y, y' = y (horizontal shear) shear_y: x' = x, y' = s·x + y (vertical shear) Alternative: shear_x(φ) uses tan(φ) — tilts by angle φ from vertical axis

7.1.3 — Rotation

rotate(φ) = [cos(φ) −sin(φ)] [sin(φ) cos(φ)] Derivation: rotate point (r·cos α, r·sin α) by φ: x' = r·cos(α+φ) = x·cos(φ) − y·sin(φ) y' = r·sin(α+φ) = y·cos(φ) + x·sin(φ) φ > 0: counter-clockwise rotation

7.1.4 — Reflection

reflect_y (across y-axis) = [−1 0] x' = −x, y' = y [ 0 1] reflect_x (across x-axis) = [ 1 0] x' = x, y' = −y [ 0 −1] Reflection = scaling with a negative scale factor (−1) on one axis.
🎯
Exam type question

Given a vertex and a transformation matrix, calculate the resulting vertex. Practice applying each matrix type by hand.

Section 7.3 — Translation & Homogeneous Coordinates

Translation cannot be expressed as a 2×2 or 3×3 linear transformation — it's not linear (it doesn't preserve the origin). The solution is homogeneous coordinates: add an extra dimension.

2D Homogeneous Coordinates

2D point (x, y) becomes (x, y, 1) in homogeneous coords. Translation matrix (3×3): T(dx, dy) = [1 0 dx] Result: x' = x + dx [0 1 dy] y' = y + dy [0 0 1] All 2D transforms as 3×3: scale: [sx 0 0] rotate: [cos φ −sin φ 0] [ 0 sy 0] [sin φ cos φ 0] [ 0 0 1] [ 0 0 1]

3D Translation (4×4 homogeneous)

3D point (x, y, z) becomes (x, y, z, 1). T(dx, dy, dz) = [1 0 0 dx] [0 1 0 dy] [0 0 1 dz] [0 0 0 1]
💡
Why homogeneous coordinates?

They allow ALL transformations (including translation) to be expressed as matrix multiplication. This means the full model-view-projection pipeline is just a sequence of matrix multiplications — exactly what the GPU does.

Section 7.2 — 3D Linear Transformations

3D Scale

scale(sx, sy, sz) = [sx 0 0 ] [ 0 sy 0 ] [ 0 0 sz]

3D Rotation

Rotate about Z-axis: Rotate about X-axis: [cos φ −sin φ 0] [1 0 0 ] [sin φ cos φ 0] [0 cos φ −sin φ] [ 0 0 1] [0 sin φ cos φ] Rotate about Y-axis: [ cos φ 0 sin φ] [ 0 1 0 ] [−sin φ 0 cos φ] Note: Y-axis rotation has sin signs swapped — remember this!

Rotation About an Arbitrary Axis (composed)

To rotate an object about its own centre (not the world origin):

  1. Translate the object so its centre is at the world origin: T(−centre)
  2. Apply the rotation matrix: R(φ)
  3. Translate back: T(+centre)
M_total = T(+centre) · R(φ) · T(−centre) Example: sphere at (1, 0.5, −0.25), rotate 25° about its x-axis: Step 1: T(−1, −0.5, 0.25) // move centre to origin Step 2: Rx(25°) // rotate Step 3: T(+1, +0.5, −0.25) // move back M = T(+c) · Rx(25°) · T(−c)
🎯
Matrix composition order matters

Transformations are applied right-to-left: M = A·B·C applies C first, then B, then A. Always construct the matrices in the correct order.

7.2.2 — Transforming Normal Vectors

Normal vectors do not transform the same way as regular points/vertices. Using the regular transform M on a normal gives an incorrect result (normal will no longer be perpendicular to the surface).

Correct normal transform: n_new = (M⁻¹)ᵀ · n Where: M = the transformation matrix applied to vertices M⁻¹ = inverse of M ᵀ = transpose n = original normal (unit vector) After transforming, re-normalize n_new for lighting calculations.

Section 7.4 — Inverse Transformations

Two options for undoing a transformation:

OptionMethodExample
Option A Compute the matrix inverse: M⁻¹ such that M·M⁻¹ = I General — works for any invertible matrix
Option B Construct an "undo" matrix manually T(d) → undo with T(−d)
Rx(45°) → undo with Rx(−45°)

Practical Example

GLSL (used in vertex shader, Ch 17)
// Vertex shader applies transformation matrices
#version 330 core
layout(location = 0) in vec3 in_Position;
layout(location = 1) in vec3 in_Normal;

uniform mat4 model;       // object → world space
uniform mat4 view;        // world → camera space
uniform mat4 projection;  // camera → clip space
uniform mat3 normalMatrix; // = transpose(inverse(model))

out vec3 fragNormal;

void main() {
    // Apply model-view-projection (right to left)
    gl_Position = projection * view * model * vec4(in_Position, 1.0);

    // Transform normal correctly
    fragNormal = normalize(normalMatrix * in_Normal);
}
